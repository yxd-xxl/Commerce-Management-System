# 开发说明文档

> **📱 移动端优先项目。** 网页端使用请按 `F12` → 开发者工具 → 切换移动端视图，推荐 iPhone 12 Pro (390×844)。TabBar、工具宫格、卡片布局均为移动端尺寸设计。

## 1. 本次目标
在不推翻原项目结构的前提下，彻底废除旧首页 UI，重做为“校园商会运营驾驶舱（Campus Ops Cockpit）”，并保持：
- Pinia 状态管理
- API 分层
- hooks/composables 抽离
- types 保留
- SCSS 变量体系
- 页面可运行、可构建、可交接

## 2. 架构原则
统一数据链路：
`page -> hook -> store -> api -> mock`

执行方式：
- `HomePage.vue` 只负责组合布局与事件分发
- 业务模块通过 `props + emits` 通信
- 组件不直接读取 mock/store、不直接发请求

1. 完整替换旧首页 `data-row/content-row` 卡片墙模型  
2. 新增运营地图（`OpsOrbitMap`）实现中心指数 + 环形节点  
3. 新增行动指令栏（`ActionCommandRail`）按优先级分组  
4. 新增项目推进流与公告脉冲条，模块展示方式差异化  
5. 首页加载替换为 `ChamberLoadingStage`，不再使用普通圆圈加载

## 5. 数据与功能联动
`homeDashboard` 统一提供：
- `user / chamber / summary / priorityTodo`
- `quickActions / metrics / activities / projects / notices / timeline`

功能联动：
- 搜索：`/pages/search/index?keyword=...`
- 今日重点：优先跳 `priorityTodo.targetPath`
- 运营地图节点：分别跳会员、项目、活动、公告、财务等页面
- 行动指令栏：按 action path 跳转
- 项目流/公告/时间线：按条目 path 跳转
